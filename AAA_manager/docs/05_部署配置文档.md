# 部署配置文档

> 版本：v1.0 | 最后更新：2026-05-17 | 状态：初稿

## 1. 环境要求

### 1.1 基础环境
- **操作系统**：Windows 10/11、Linux、macOS
- **Python**：3.11+
- **包管理**：conda（推荐）或 pip + virtualenv
- **Git**：2.30+（CLI工作流依赖）

### 1.2 外部服务依赖
| 服务 | 必需 | 用途 |
|------|------|------|
| DeepSeek API | 是 | 主LLM提供商 |
| Qwen API | 否 | 备用LLM提供商 |
| 讯飞ASR API | 否 | 语音识别（可选） |
| 网络搜索API | 否 | 岗位信息搜索（可选） |

## 2. 安装部署

### 2.1 创建conda环境
```bash
conda create -n interview_exp python=3.11 -y
conda activate interview_exp
```

### 2.2 安装依赖
```bash
cd AAA_manager
pip install -r requirements.txt
```

**requirements.txt 依赖清单**：
| 包名 | 用途 |
|------|------|
| openai | LLM API调用（兼容DeepSeek/Qwen） |
| gitpython | Git操作封装 |
| python-dotenv | 环境变量加载 |
| fastapi | Web框架 |
| uvicorn | ASGI服务器 |
| pdfplumber | PDF简历解析 |
| openpyxl | Excel读取 |
| httpx | 异步HTTP客户端 |
| websockets | WebSocket支持 |
| pytest | 测试框架 |
| pytest-asyncio | 异步测试支持 |
| pytest-mock | Mock工具 |
| httpx[http2] | HTTP/2支持 |

### 2.3 配置环境变量
```bash
# 复制模板
cp .env.example .env

# 编辑 .env 填入实际配置
```

## 3. 环境变量配置详解

### 3.1 LLM配置（必填）

| 变量名 | 说明 | 示例值 |
|--------|------|--------|
| DEEPSEEK_API_KEY | DeepSeek API密钥 | `sk-xxx` |
| DEEPSEEK_BASE_URL | DeepSeek API地址 | `https://api.deepseek.com` |
| DEEPSEEK_MODEL | DeepSeek模型名 | `deepseek-chat` |
| QWEN_API_KEY | Qwen API密钥（备用） | `sk-xxx` |
| QWEN_BASE_URL | Qwen API地址 | `https://dashscope.aliyuncs.com/compatible-mode/v1` |
| QWEN_MODEL | Qwen模型名 | `qwen-plus` |
| ACTIVE_PROVIDER | 当前激活的提供商 | `deepseek` 或 `qwen` |

### 3.2 路径配置

| 变量名 | 说明 | 默认值 |
|--------|------|--------|
| INTERVIEW_REPO_PATH | 面试仓库根目录（英文路径） | 项目父目录 |
| RAW_QUESTIONS_DIR | 面试原始问题目录名 | `面试原始问题` |
| REVIEW_DIR | 面试复盘目录名 | `面试复盘` |
| QUESTION_BANK_DIR | 问题库目录名 | `问题库` |
| PREDICTION_DIR | 岗位预测目录名 | `岗位预测` |
| RESUME_DIR | 简历目录名 | `个人情况/简历` |

### 3.3 功能开关

| 变量名 | 说明 | 默认值 |
|--------|------|--------|
| GIT_ENABLED | 启用Git操作 | `true` |
| WEB_SEARCH_ENABLED | 启用网络搜索 | `false` |
| FOLLOWUP_ENABLED | 启用追问预测 | `true` |
| ASR_ENABLED | 启用语音识别 | `false` |

### 3.4 Web服务配置

| 变量名 | 说明 | 默认值 |
|--------|------|--------|
| WEB_HOST | 监听地址 | `0.0.0.0` |
| WEB_PORT | 监听端口 | `8000` |

### 3.5 网络搜索配置（可选）

| 变量名 | 说明 |
|--------|------|
| SEARCH_API_KEY | 搜索API密钥 |
| SEARCH_API_URL | 搜索API地址 |

### 3.6 讯飞ASR配置（可选）

| 变量名 | 说明 |
|--------|------|
| XUNFEI_APP_ID | 讯飞应用ID |
| XUNFEI_API_KEY | 讯飞API Key |
| XUNFEI_API_SECRET | 讯飞API Secret |

### 3.7 项目文档配置（可选）

| 变量名 | 说明 |
|--------|------|
| PROJECT_CONFIGS | 项目文档路径配置字符串，格式：`项目名:路径:文件1,文件2;...` |

## 4. 启动方式

### 4.1 Web服务
```bash
cd AAA_manager
python app.py
```
服务启动后访问 `http://localhost:8000`。

### 4.2 CLI工具
```bash
cd AAA_manager

# 全流程同步
python main.py sync

# 带自动提交
python main.py sync --auto-commit

# 单独检测变更
python main.py detect

# 单独抽取问题
python main.py extract <文件路径>

# 单独归档
python main.py archive <文件路径>

# 单独复盘
python main.py review <文件路径>

# 岗位预测
python main.py prepare --company 公司名 --position 岗位名
```

## 5. 目录结构要求

项目运行时依赖以下外部目录结构（位于INTERVIEW_REPO_PATH下）：

```
{INTERVIEW_REPO_PATH}/
├── 面试原始问题/      # 放入面试记录.md文件
├── 面试复盘/          # 自动生成复盘报告
├── 问题库/            # 自动维护的题库
├── 岗位预测/          # 预测题输出
├── 个人情况/简历/     # 放入简历PDF
└── 公司投递情况/      # 放入投递Excel
```

> **重要**：INTERVIEW_REPO_PATH 必须使用英文路径，避免中文路径导致的编码问题。

## 6. 数据目录

`AAA_manager/data/` 目录存放运行时数据：

```
data/
├── sessions/           # 会话JSON文件（自动创建）
│   └── {session_id}.json
└── user_profile.json   # 用户画像（自动生成）
```

## 7. 日志配置

日志输出到 `AAA_manager/logs/` 目录：
- 控制台：彩色输出，INFO级别
- 文件：`logs/app.log`，按大小轮转（5MB/文件，保留3个备份）
- 同步日志：`logs/sync_{date}.log`，按日期分文件

## 8. 常见问题

### Q: Git操作报错
**A**：确认 `GIT_ENABLED=true` 时，INTERVIEW_REPO_PATH 指向的目录是有效的Git仓库。若不需要Git功能，设置 `GIT_ENABLED=false`。

### Q: LLM调用失败
**A**：检查API Key是否正确配置。系统内置重试机制（3次，指数退避2s/4s/8s），若仍失败请检查网络连通性。

### Q: 中文路径问题
**A**：INTERVIEW_REPO_PATH 必须使用英文路径。建议将面试仓库放在纯英文路径下。

### Q: PDF解析失败
**A**：确认简历文件为标准PDF格式（非扫描件图片PDF），pdfplumber 仅支持文本型PDF。

### Q: conda环境名称
**A**：项目统一使用 `interview_exp` 作为conda环境名。
