# 系统架构设计文档

> 版本：v1.1 | 最后更新：2026-05-17 | 状态：已更新

## 1. 架构概览

AI面试准备助手（AAA_manager）采用**双工作流混合架构**，同时提供 Web 服务和 CLI 工具两种交互方式，共享底层核心模块。

```
┌─────────────────────────────────────────────────────────────┐
│                      用户交互层                               │
├─────────────────────┬───────────────────────────────────────┤
│   Web 服务 (FastAPI) │         CLI 工具 (argparse)            │
│   - 模拟面试问答     │         - sync 全流程同步              │
│   - 用户画像         │         - reflect 面试反思交互         │
│   - 实时SSE流式      │         - detect 变更检测              │
│   - 实时SSE流式      │         - extract 问题抽取             │
│   - 语音输入(ASR)    │         - archive 问题归档             │
│                     │         - review 复盘分析               │
│                     │         - prepare 岗位预测              │
│                     │         - export-session 会话导出       │
├─────────────────────┴───────────────────────────────────────┤
│                      核心业务层                               │
├─────────────────────────────────────────────────────────────┤
│  LLM Client │ Knowledge │ Profile │ Git Ops │ Web Searcher  │
├─────────────────────────────────────────────────────────────┤
│                      基础设施层                               │
├─────────────────────────────────────────────────────────────┤
│  Config │ Logger │ Data Storage (JSON/Markdown)              │
└─────────────────────────────────────────────────────────────┘
```

## 2. 模块划分

### 2.1 用户交互层

| 模块 | 入口文件 | 职责 |
|------|---------|------|
| Web服务 | `app.py` | FastAPI应用启动、路由注册、CORS、中间件 |
| API路由 | `api/routes/*.py` | 各功能端点实现 |
| 前端界面 | `frontend/static/` | 三栏布局Web UI |
| CLI工具 | `main.py` | 命令行交互、子命令分发 |

### 2.2 核心业务层

| 模块 | 文件 | 职责 |
|------|------|------|
| 变更检测器 | `detector.py` | 扫描面试原始问题目录，识别新增/变更文件 |
| 问题抽取器 | `extractor.py` | 通过LLM从非结构化面试记录中抽取问题 |
| 问题归档器 | `archiver.py` | 分类问题并追加到对应题库文件 |
| 复盘生成器 | `reviewer.py` | 生成面试复盘分析报告（可选集成反思上下文） |
| 反思交互器 | `reflector.py` | 多轮对话收集实际回答表现（PydanticAI 双 Agent + 渐进式披露） |
| 岗位预测器 | `preparer.py` | 结合JD/简历/项目生成预测题 |
| 会话导出器 | `exporter.py` | 从模拟面试会话中导出问题 |
| Git操作 | `git_ops.py` | 文件暂存、提交、信息生成 |

### 2.3 知识库层

| 模块 | 文件 | 职责 |
|------|------|------|
| 问题库引擎 | `knowledge/question_bank.py` | 多维度检索、分类过滤、加权排序 |
| 项目文档读取 | `knowledge/project_reader.py` | 四层渐进式项目理解 |
| 简历解析 | `knowledge/resume_reader.py` | PDF简历文本提取 |
| Excel处理 | `knowledge/excel_reader.py` | 投递情况表格读取与颜色状态映射 |

### 2.4 AI集成层

| 模块 | 文件 | 职责 |
|------|------|------|
| LLM客户端 | `llm_client.py` | OpenAI SDK封装，支持DeepSeek/Qwen切换 |
| 反思Agent | `reflector.py` | PydanticAI 双 Agent（Conversation + Summary），渐进式披露项目文档 |
| 网络搜索 | `core/web_searcher.py` | 联网搜索公司/技术信息 |
| 岗位预测Agent | `core/prepare_agent.py` | 手写ReAct Loop，基于OpenAI function calling自主决策出题流程 |
| ASR语音 | `core/asr_xunfei.py` | 讯飞语音识别集成 |
| 提示词系统 | `prompts/*.md` | 各场景Prompt模板 |

### 2.5 用户画像层

| 模块 | 文件 | 职责 |
|------|------|------|
| 画像管理器 | `profile/profile_manager.py` | 综合画像构建、增量更新、摘要生成 |

### 2.6 基础设施层

| 模块 | 文件 | 职责 |
|------|------|------|
| 配置管理 | `config.py` | 环境变量加载、路径管理、功能开关 |
| 日志系统 | `logger.py` | 彩色控制台+文件轮转日志 |
| 数据存储 | `data/` | JSON会话文件、用户画像持久化 |

## 3. 数据流设计

### 3.1 CLI全流程同步（sync）

**标准模式**：

```
面试原始问题/*.md
       │
       ▼
  ┌─────────┐     ┌──────────┐     ┌──────────┐     ┌──────────┐
  │ Detector │────▶│ Extractor │────▶│ Archiver │────▶│ Reviewer │
  │ 变更检测  │     │ 问题抽取   │     │ 分类归档  │     │ 复盘生成  │
  └─────────┘     └──────────┘     └──────────┘     └──────────┘
       │                                  │                │
       ▼                                  ▼                ▼
  .last_sync_time              问题库/*.md更新      面试复盘/*.md
```

**反射增强模式（--reflect）**：

```
面试原始问题/*.md
       │
       ▼
  ┌─────────┐     ┌──────────┐     ┌──────────┐     ┌──────────┐     ┌──────────┐
  │ Detector │────▶│ Extractor │────▶│ Reflector│────▶│ Reviewer │────▶│ Archiver │
  │ 变更检测  │     │ 问题抽取   │     │ 反思交互  │     │ 复盘增强  │     │ 分类归档  │
  └─────────┘     └──────────┘     └──────────┘     └──────────┘     └──────────┘
       │                                  │                │                │
       ▼                                  ▼                ▼                ▼
  .last_sync_time              user_profile.json  面试复盘/*.md   问题库/*.md更新
                                (画像更新)        (含反思上下文)
```

### 3.2 Web问答流程

```
用户输入问题
       │
       ▼
  ┌──────────────┐
  │ 项目别名检测   │ ← PROJECT_ALIASES
  └──────┬───────┘
         ▼
  ┌──────────────┐
  │ QuestionBank │ ← 问题库 + 岗位预测题
  │  多维度检索   │
  └──────┬───────┘
         ▼
  ┌──────────────┐
  │  LLM生成回答  │ ← 检索结果 + 画像摘要 + 对话历史
  └──────┬───────┘
         ▼
  SSE流式响应 → 前端渲染
```

### 3.3 用户画像构建

```
简历PDF + 面试历史 + Excel投递表
              │
              ▼
     ┌────────────────┐
     │ ProfileManager │
     │ LLM综合分析     │
     └────────┬───────┘
              ▼
   data/user_profile.json
   (技能图谱/优劣势/趋势/建议)
```

## 5. 关键设计决策

### 5.1 双工作流架构
- **Web服务**：面向日常使用，提供实时问答和画像服务
- **CLI工具**：面向数据整理，支持各环节独立运行或流水线串联
- **共享核心**：两条工作流共享 LLM、知识库、配置等底层模块

### 5.2 数据存储策略
- **Markdown文件**：问题库、面试记录、复盘报告（便于Git版本控制和人工编辑）
- **JSON文件**：会话历史、用户画像（结构化数据，便于程序读写）
- **无数据库依赖**：全文件化存储，零运维成本

### 5.3 LLM集成策略
- 基于OpenAI SDK统一封装，支持DeepSeek/Qwen双提供商热切换
- 同步+流式双接口，CLI用同步，Web用流式
- 内置重试机制（指数退避），容错性强
- 各场景通过独立Prompt模板解耦
- **岗位预测走 Agent 路径**：手写 ReAct Loop + function calling，LLM 自主决策工具调用（搜索/读项目/出题/自评），失败时 fallback 到精简线性流水线

### 5.4 知识检索机制
- QuestionBank：多阶段评分（精确匹配→正文匹配→n-gram→字符重叠）
- 项目别名加权：识别问题中的项目关键词，boost对应category分数
- 岗位预测题自动纳入检索范围，与主题库统一接口

### 5.5 可选功能开关
- 通过 `.env` 环境变量控制功能启用/禁用
- GIT_ENABLED：Git操作开关
- WEB_SEARCH_ENABLED：网络搜索开关
- FOLLOWUP_ENABLED：追问生成开关
- ASR_ENABLED：语音识别开关

## 6. 扩展性设计

- **新增LLM提供商**：只需在 `config.py` 添加配置，`llm_client.py` 无需修改
- **新增题库分类**：在问题库目录添加对应 .md 文件即自动识别
- **新增项目文档**：通过 PROJECT_CONFIGS 或自动发现机制注册
- **新增API端点**：在 `api/routes/` 添加路由文件并在 `app.py` 注册
- **新增CLI命令**：在 `main.py` 添加子命令解析和处理函数
