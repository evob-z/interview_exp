# 项目需求文档

> 版本：v1.1 | 最后更新：2026-05-17 | 状态：已更新

## 1. 项目概述

### 1.1 项目背景
求职面试准备过程中，候选人面临大量分散的面试记录、题目整理、复盘分析等工作，手动管理效率低、容易遗漏。本项目旨在构建一个AI驱动的面试准备助手，自动化面试数据的整理归档，并提供智能问答辅助备战。

### 1.2 项目目标
- 自动化面试记录的结构化整理与归档
- 提供AI辅助的模拟面试问答能力
- 构建可检索的个人面试题库资产
- 生成面试复盘分析帮助持续改进
- 支持针对性岗位备战预测题生成

### 1.3 目标用户
求职中的技术岗位候选人（个人使用），需要系统性地管理面试准备材料。

## 2. 功能需求

### 2.1 面试题库管理

#### FR-001 分类题库维护
- **描述**：按分类管理面试题目，支持项目题、八股题、AI编程题等多种分类
- **输入**：结构化的面试题目（含答题要点）
- **输出**：按分类组织的Markdown题库文件
- **业务规则**：
  - 入库时自动去重（相似度>80%判定为重复）
  - 每道题包含题目和LLM生成的答题要点
  - 分类对应问题库目录下的独立.md文件

#### FR-002 题库多维检索
- **描述**：支持关键词搜索、分类过滤、项目加权等多维度检索
- **输入**：搜索关键词、可选的分类/项目过滤条件
- **输出**：按相关度排序的匹配题目列表
- **业务规则**：
  - 多阶段评分：精确匹配→正文匹配→n-gram相似度
  - 项目别名自动映射到规范category
  - 岗位预测题与主题库统一检索接口

### 2.2 面试记录自动处理

#### FR-003 变更检测
- **描述**：自动扫描面试原始问题目录，识别新增的面试记录文件
- **输入**：面试原始问题目录
- **输出**：新增文件列表（区分面试记录、问题库变更等）
- **业务规则**：
  - 支持Git变更检测和文件系统时间戳检测双模式
  - 通过GIT_ENABLED开关控制
  - 记录最后同步时间点

#### FR-004 智能问题抽取
- **描述**：从非结构化面试记录中，通过LLM智能识别和抽取结构化问题列表
- **输入**：面试记录Markdown文件
- **输出**：结构化问题列表（含公司、轮次、问题内容）
- **业务规则**：
  - 自动识别输入格式（录音转写/口语笔记/已结构化）
  - 抽取时进行指代消解（如"你那个项目"→具体项目名）
  - 支持强制文件类型参数覆盖自动检测

#### FR-005 分类归档
- **描述**：将抽取的问题按LLM建议的分类追加到对应题库文件
- **输入**：结构化问题列表 + 分类建议
- **输出**：更新后的题库文件
- **业务规则**：
  - 入库标题必须符合规定格式
  - 去重检测避免重复入库
  - LLM生成答题要点一并存入

#### FR-006 面试复盘分析
- **描述**：对面试记录生成深度复盘报告
- **输入**：面试记录文件
- **输出**：结构化复盘报告（独立文件或追加到原文件）
- **业务规则**：
  - 包含问题分类占比、面试官关注点、面试官画像
  - 包含改进建议和鼓励
  - 支持standalone模式生成独立复盘文件
  - 支持通过反射上下文增强复盘质量（--reflect 模式）

#### FR-006b 面试反思交互
- **描述**：通过多轮终端对话收集候选人对面试的实际回答表现，LLM 自主提问直到信息覆盖度达标
- **输入**：面试记录文件 + 用户画像 + 岗位预测（可选）+ 项目文档索引
- **输出**：结构化反思报告（含表现总结、优缺点、面试官关注点、改进建议）
- **业务规则**：
  - 使用 PydanticAI 双 Agent 架构（Conversation Agent + Summary Agent）
  - Conversation Agent 按 5 维度覆盖度评分动态提问（overall_feeling / strengths / weaknesses / interviewer_focus / improvement_areas）
  - 停止策略：全部维度 ≥70 自动停止 / 8 轮硬上限 / 用户输入 `/stop` 手动停止
  - 项目文档渐进式披露：首轮仅注入文件清单，Agent 通过 tools 按需查阅
  - 反思成功后自动更新用户画像（strengths / weaknesses）
  - 反思失败时不影响后续 review/archive 正常完成

### 2.3 模拟面试问答

#### FR-007 多模式智能问答
- **描述**：提供模拟面试、详细讲解、快速回答三种问答模式
- **输入**：用户问题 + 对话模式
- **输出**：AI生成的回答（SSE流式）
- **业务规则**：
  - 模拟面试模式：简洁话术风格，模拟真实面试场景
  - 详细讲解模式：深入展开原理和示例
  - 快速回答模式：给出核心要点
  - 自动检索题库相关内容作为上下文

#### FR-008 追问预测（可选）
- **描述**：基于当前对话生成可能的追问方向
- **输入**：当前对话历史
- **输出**：3条预测追问
- **业务规则**：通过FOLLOWUP_ENABLED开关控制

#### FR-009 会话历史管理
- **描述**：支持多会话的保存、切换和删除
- **输入/输出**：会话JSON文件（data/sessions/）
- **业务规则**：
  - 每个会话独立JSON文件
  - 支持新建对话、切换会话、删除历史

### 2.4 用户画像

#### FR-010 综合画像构建
- **描述**：综合简历、面试记录、投递情况构建候选人画像
- **输入**：简历PDF + 面试历史 + Excel投递表
- **输出**：结构化画像（技能图谱/优劣势/趋势/建议）
- **业务规则**：
  - LLM分析优先，不可用时降级为纯统计
  - 支持增量更新（每次面试后追加）
  - 画像摘要化（4-6句）用于问答上下文

#### FR-011 个性化建议与鼓励
- **描述**：基于画像生成面试准备建议和鼓励话语
- **输入**：用户画像数据
- **输出**：个性化建议文本

### 2.5 岗位针对性备战

#### FR-012 岗位预测题生成（Agent 驱动）
- **描述**：针对目标岗位，Agent 自主决策：联网搜 JD → 读简历/项目 → 结合用户画像 → 生成预测题 → 自评质量 → 迭代优化
- **输入**：公司名、岗位名、可选部门名、可选面试日期
- **输出**：12-20 道预测题（写入 `岗位预测/` 目录，自动纳入模拟面试检索）
- **业务规则**：
  - Agent 默认启用（手写 ReAct Loop + OpenAI function calling，零额外框架依赖）
  - Agent 自主判断 JD 样本量：`total_snippets_so_far < 3` 时自动换关键词补充搜索
  - 生成后 LLM 自评四维度（jd_coverage / project_depth / profile_match / dup_rate），`overall < 0.6` 自动迭代
  - Agent 异常时若 `PREP_AGENT_FALLBACK=true` 自动回退线性流水线（单次 LLM 出题）
  - 支持 `--department` 参数（如 `CHO体系-企业信息化部`），注入题目生成上下文
  - 同岗位历史预测题去重提醒

### 2.6 数据导入导出

#### FR-013 会话问题导出
- **描述**：从模拟面试会话中提取原始问题，输出为Markdown文件
- **输入**：会话ID
- **输出**：面试原始问题目录下的.md文件
- **业务规则**：
  - 默认启用LLM上下文感知改写
  - 改写后的问题自包含且带项目标注

#### FR-014 投递情况Excel读取
- **描述**：读取投递情况Excel表格，按颜色识别投递状态
- **输入**：Excel文件
- **输出**：结构化投递数据（公司、状态、筛选结果）
- **业务规则**：按背景颜色映射状态（已结束/offer/无消息/流程中/已投递）

### 2.7 语音输入（可选）

#### FR-015 语音转文字
- **描述**：通过讯飞ASR API实现语音输入
- **输入**：音频流
- **输出**：识别文本
- **业务规则**：通过ASR_ENABLED开关控制

## 3. 非功能需求

### 3.1 性能需求
- Web问答响应：首token < 2s（依赖LLM API延迟）
- CLI sync全流程：单文件处理 < 30s
- 题库检索：< 500ms

### 3.2 可用性需求
- 零数据库依赖，纯文件存储
- 核心功能离线可用（仅LLM调用需网络）
- 可选功能通过开关控制，不影响核心流程

### 3.3 可维护性需求
- 模块化设计，各组件可独立测试
- 统一日志系统，支持问题排查
- 配置外部化，通过.env管理

### 3.4 数据安全
- 敏感信息（API Key、个人简历）不入Git
- .env文件本地管理，提供.env.example模板
- 会话数据本地存储

## 4. 用户场景

### 场景1：面试后快速整理
1. 用户将面试记录（语音转写/手记）保存为.md文件到`面试原始问题/`目录
2. 运行 `python main.py sync` 触发全流程
3. 系统自动检测新文件 → 抽取问题 → 分类入库 → 生成复盘
4. 用户查看复盘报告，了解自己的表现分析和改进建议

### 场景2：日常模拟面试
1. 用户打开Web界面 (`python app.py`)
2. 选择"模拟面试"模式
3. 输入问题或选择题库中的题目
4. 系统结合题库知识给出面试话术风格的回答
5. 查看追问预测，继续深入练习

### 场景3：岗位定向备战
1. 用户即将面试某公司某岗位
2. 运行 `python main.py prepare --company 字节跳动 --position AI安全`
3. 系统联网搜索JD，结合简历和项目生成预测题
4. 预测题自动可被模拟面试功能检索到
5. 用户在Web端针对预测题逐一练习

### 场景4：查看个人成长
1. 用户在Web端点击"画像"面板
2. 系统综合所有面试数据展示技能图谱、优劣势分析
3. 对比历次面试表现，展示成长趋势
4. 获取个性化改进建议和鼓励

### 场景5：批量数据同步
1. 用户积累了多份面试记录未处理
2. 运行 `python main.py sync --auto-commit`
3. 系统批量检测、处理所有新文件
4. 自动Git提交整理结果
